# 환경 레이어 복원 설계안 — 유실된 효과 디스패처 복구 + env(칩/회복/해저드) 데이터 인코딩

> 풀배틀 resync(C2) 후 라운드별로 격리된 잔차를 분석한 결과, 지배적 잔차는 **환경 레이어**였다.
> 트레이스 데이터가 이를 가리켰고, 그 과정에서 **핸드오프가 엔진 능력을 과대 기술**했음을
> 발견했다. 본 설계안은 그 현실 위에서 효과-스키마(키스톤)를 복원·확장하는 로드맵이다.

## 0. 발견 (데이터로 확정)

- **엔진에 일반 효과 디스패처(PR-E1/E2)도 날씨도 없다.** Grep 확인: `engine.py`에 `weather`/
  `rain`/`sandstorm` 문자열 0건, `_act_effect_dispatch`/`_eff_cond_ok` 0건. 핸드오프
  (트레이스리플레이)가 "PR-E1/E2 전부 적용"·"엔진 날씨 인프라(F3) 있음"이라 했으나 현재
  엔진과 맞지 않음 — B3~C2 엔진 정리("일부 정리됨") 과정에서 유실된 것으로 보인다.
- 엔진이 실제로 가진 것: *단발 메커니즘 슬롯* `mechanisms['leftovers']`·`['status']`·
  `['protean']`·`['hazard']`(전부 gimmick_col/match_value로 **하나**만 매칭) + `move_effects`
  (무브명별 스탯 부스트/해저드 설치) + trace_actions/preserve_ids/resync.
- **배선이 끊겨 있다**: `battle_setup.build_battle_spec`가 `mechanisms['effects']=EFFECTS`를
  주입하지만(71줄) 엔진엔 그걸 소비할 디스패처가 없다 → gen5 풀배틀에서 env가 하나도 안 돈다.
- 결과: run_b4 ★ 행 다수가 "엔진 0" — Leftovers 회복(+25/+26)·Stealth Rock(−33~−95)·모래
  (−19)·독/화상 틱·Rough Skin·Rocky Helmet(T5 −52/−70)이 전부 미모델. resync가 누적은 막아
  desync는 없지만, 라운드별 데미지 충실도가 비는 자리.

## 1. env demand 카탈로그 (골든 gen5 로그 env 스트림 실측)

트레이스 env 스트림에서 직접 추출(showdown_trace). 수치는 maxHP 분수로 확정:

| 메커니즘 | 출처 | 트리거 | 효과 | 조건 | 스코프 |
|---|---|---|---|---|---|
| Leftovers | item | ON_TURN_END | heal_frac 1/16 | — | self |
| Black Sludge | item | ON_TURN_END | heal_frac 1/16(Poison) / damage_frac 1/8(비Poison) | self type | self |
| Rain Dish | ability | ON_TURN_END | heal_frac 1/16 | weather=rain | self |
| sandstorm chip | weather | ON_TURN_END | damage_frac 1/16 | weather=sand & not(Rock/Ground/Steel) | self |
| 독(psn) | status | ON_TURN_END | damage_frac 1/8 | status=psn | self |
| 맹독(tox) | status | ON_TURN_END | damage_frac n/16 누진 | status=tox | self |
| 화상(brn) | status | ON_TURN_END | damage_frac 1/8 | status=brn | self |
| Stealth Rock | hazard | ON_SWITCH(진입) | damage_frac (Rock상성×1/8) | 진영에 SR | 진입자 |
| Rough Skin / Iron Barbs | ability | ON_HIT | damage_frac 1/8 | contact | attacker |
| Rocky Helmet | item | ON_HIT | damage_frac 1/6 | contact | attacker |

실측 정합 근거(env 덤프): Garchomp(Rikimaru, maxhp 420) Rough Skin −52(=⌊420/8⌋)·Rocky Helmet
−70(=⌊420/6⌋); Leftovers +25/+26(=1/16); 모래 −19(Latios 304/16); SR Latios −38(Dragon/Psychic
중립 1/8=38)·Zapdos −95(Flying ×2 → 1/4)·Bonaparte −40.

## 2. 통합 shape (유실 디스패처 + 확장)

복구할 디스패처(유실 PR-E1, 코드 보존됨)의 shape는 demand를 이미 덮는다:

```
effects = { "<ability/item/status/weather 이름>": {
    "trigger":  "ON_HIT | ON_TURN_END | ON_SWITCH",
    "condition":{...},     # contact / weather:<x> / status:<x> / not_types:[...]
    "effect":   {"type":"damage_frac|heal_frac", "frac":<f>, "of":"maxhp"},
    "scope":    "self|attacker|target",
    "source":   "ability|item|status|weather|hazard",
}, ... }
```

확장 포인트(다중성 — 단발 슬롯이 못 하던 것):
- **소유자 다중**: 디스패처가 *관련 캐릭터들*의 ability/item/status를 각각 매칭 → 여러 Leftovers
  보유자·개별 상태이상이 동시에 처리됨(단발 슬롯은 하나만).
- **ON_TURN_END 스코프**: 소유자=그 턴 active_char(턴루프가 유닛마다 post-target에서 호출).
- **조건 확장**: 기존 `contact`에 `weather:<x>`·`status:<x>`·`not_types:[...]`(모래 면역) 추가.

## 3. 상태 공급 (트레이스 관측값 → 엔진)

트레이스는 관측 상태를 이미 담는다(검증됨):
- **날씨**: move 이벤트 `context.weather`(RainDance/Sandstorm) + `-weather` 전이. → 라운드별
  발효 날씨를 `field_state['weather']`에 공급(resync round-start 훅 재사용).
- **상태이상**: HP 접미사 tox/brn/psn 파싱(showdown_trace). → 진입/부여 시 char에 status 세팅.
- **해저드**: env `Stealth Rock` + move `Stealth Rock` 설치. → `field_state['hazard']` 진영별.

resync(C2)가 이미 라운드 시작에 HP·on_field를 로그로 박는다 — 여기에 weather/status/hazard
공급을 얹으면 디스패처 조건이 관측 상태로 평가된다. *상태 자체를 시뮬하지 않고 관측값을
주입*하는 트레이스-리플레이 원칙 그대로.

## 4. PR 시퀀스 (작게·검증가능·레버리지순 절충)

- **PR-E′1 — 디스패처 코어 복원 + ON_HIT 접촉** (자기완결, 클린룸 검증됨):
  유실된 `_act_effect_dispatch`/`_eff_*` 4함수 복구 + `_act_on_hit` 끝 호출. reference_gen5에
  EFFECTS(dict) + CONTACT_MOVES + Garchomp 세트 교정(Rough Skin/Rocky Helmet — env 역설계).
  battle_setup가 char에 ability/item 노출 + 무브에 contact 플래그. **키스톤 재건**.
  검증: Garchomp/Hippowdon 420→368→298(클린룸 통과). 회귀0: effects 미설정/ability없음 no-op.
- **PR-E′2 — ON_TURN_END heal_frac/damage_frac + 상태 공급**:
  디스패처를 ON_TURN_END에 등록 + heal_frac. Leftovers·Black Sludge·Rain Dish·sandstorm·
  psn/brn/tox를 EFFECTS로. 날씨/상태를 field_state·char에 공급(resync 훅). 단발 leftovers/
  status 슬롯은 미설정으로 둠(회귀0). 다수 ★행을 한 번에 닫음.
- **PR-E′3 — 진입 해저드 타입스케일(Stealth Rock)**:
  ON_SWITCH 진입 데미지를 Rock 상성으로 스케일(Tier-3 갭). 기존 `_apply_entry_hazard` 정제 또는
  디스패처 ON_SWITCH 케이스. field_state hazard 진영별.

각 PR 후 run_b4로 닫힘 확인(앱사이드), 고립 로직은 클린룸.

## 5. 정직한 경계

- **engine.py truncation 상존** — 엔진 전체 실행 검증은 앱사이드(run_b4). 디스패처 등 고립
  로직은 클린룸 standalone. Read/Grep이 진실 출처. (mnt가 *최근 수정된 .py*를 끝부분 절단해
  읽으니, 적용 후 파일은 Read로 대조하고 풀런은 앱에서.)
- **단발 슬롯은 지금 리팩터 안 함** — 디스패처가 서도 기존 leftovers/status/hazard 슬롯은
  그대로(회귀0 우선). gen5 spec은 그 슬롯을 안 쓰고 디스패처 EFFECTS만 쓴다. 흡수 통합은
  의도된 후속.
- **세트 역설계 동반** — env가 실제 ability/item을 드러낸다(Garchomp=Rough Skin+Rocky Helmet).
  EFFECTS가 발동하려면 세트가 맞아야 하므로, env가 확정한 분만 세트 교정(prior 추정 아님).
- **교차-게임 가드** — trigger×condition×effect×scope shape는 포켓몬 2세대까지만 확인. 비-포켓몬
  스탯턴제 로그로 굳히기 전 점검(잠정 결정화).
- **회귀0 게이트** — effects 미설정·char에 ability/item/status 키 없음·contact 플래그 없음이면
  디스패처 즉시 no-op. 모든 PR이 이 게이트.
