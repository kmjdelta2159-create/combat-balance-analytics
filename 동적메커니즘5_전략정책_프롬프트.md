# 동적 메커니즘 PR-F — 전략 정책 (setup_first)

Antigravity 작업 지시서. PR-E(무브 효과 경로) 위에서 작동한다. 그리디 무브 선택기를 **플러그인 가능한 정책**으로 확장한다. 기본값은 현재 그리디(회귀 0). `game_config["move_policy"] == "setup_first"`일 때만, 아직 사용하지 않은 효과 무브(`game_config["move_effects"]`에 정의된 무브 중 active_states에 그 효과가 아직 없는 것)를 데미지 무브보다 먼저 고른다. 효과를 이미 깔았으면(active_states에 marker 존재) 그리디로 데미지를 친다.

이게 전략 의사결정 모델의 첫 진짜 결정이다 — "초반에 boost를 깔고 이후 때린다"는 플레이 패턴을 재현한다. 정책 미설정 시 기존 그리디와 완전히 동일하다.

## 정직한 범위

룰 기반 단일 패턴이다(setup 1회 후 데미지). 다회 스택(Calm Mind를 2~3번)은 이 PoC 범위 밖 — 효과를 한 번 깔면 이후엔 그리디로 전환한다. HP 임계·상대 위협도·턴 수 같은 조건 확장과 ML 정책은 후속이다. 정책 식별자(move_policy)와 효과 맵(move_effects)을 채우는 입력 UI는 다음 PR(UI)이며, 본 PR은 game_config 주입으로 작동·하니스 검증된다.

## 변경 범위

`modules/engine.py`의 `_act_move_select` 1곳. movepool 가드 직후, 타겟 결정 직전에 정책 분기를 삽입한다. 그 외 영역은 건드리지 않는다.

## 적용 규칙

- FIND를 찾아 REPLACE로 그대로 교체한다. 들여쓰기·공백·한글 한 글자도 바꾸지 않는다.
- 임의 리팩터·이름 변경·하드코딩 금지. 지시된 치환만.
- 적용 후 `python -c "import modules.engine"` 통과.

# 파일: `modules/engine.py`

## MS-1 _act_move_select에 setup_first 정책 분기 추가

**FIND:**

```python
    char = ctx["active_char"]
    movepool = char.get("movepool")
    if not movepool:
        ctx["current_move"] = None
        return
    t = ctx.get("current_target")
```

**REPLACE:**

```python
    char = ctx["active_char"]
    movepool = char.get("movepool")
    if not movepool:
        ctx["current_move"] = None
        return
    # ── 전략 정책 (선택): setup_first — 아직 안 쓴 효과 무브를 데미지보다 먼저 고른다 ──
    _gc_pol = ctx.get("game_config") or {}
    if _gc_pol.get("move_policy") == "setup_first":
        _eff_map = _gc_pol.get("move_effects") or {}
        for _smv in movepool:
            _snm = _smv.get("name")
            if _snm in _eff_map and not any(
                str(_s.get("id", "")).startswith(f"move_effect_{_snm}_")
                for _s in char.get("active_states", [])
            ):
                ctx["current_move"] = _smv
                return
    t = ctx.get("current_target")
```

## 적용 후 검증 (Antigravity가 보고할 것)

1. `modules/engine.py` import 통과.
2. `grep -c 'move_policy' modules/engine.py` → 1건(정책 분기).
3. `grep -n 'setup_first' modules/engine.py` → 1건.
4. 정책 분기가 `_act_move_select` 안, movepool 가드 직후·`t = ctx.get("current_target")` 직전에 위치하는지.
5. `game_config`에 move_policy가 없을 때 기존 그리디 동작이 그대로인지(회귀 없음).
