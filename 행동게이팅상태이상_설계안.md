# 행동 게이팅 상태이상 설계안 — 갈래 ③ (마비/잠듦/혼란)

## 0. 위치
`복제완성_Trajectory_6갈래.md` §3의 독립 갈래 ③. 날씨 갈래(F3·F5) 완결 후 진입. 기존
상태이상(`mechanisms.status` = 독/화상 턴종료 데미지)과 **별개** — 이건 *행동 자체를 막는*
게이팅(마비/잠듦/혼란)이다. 보조자는 코드 직접 수정 안 함 — 정찰→설계→결정→PR→검증.

## 1. 문제 — 엔진에 "이번 턴 행동 차단"이 없다
현재 상태이상은 전부 스탯 증감(active_states) 또는 턴종료 데미지(_act_status_tick)다. "이번 턴
행동을 못 하게/자기를 치게" 하는 행동 게이팅이 없다. 마비(확률 행동불가)·잠듦(N턴 완전
행동불가)·혼란(확률 자해+행동불가)이 여기 해당.

## 2. 정찰로 확정한 배선 지점
- **행동 차단 지점**: `_act_target_select`(engine L385). 이미 `ctx["targets"]=[]; return`이
  "이번 턴 생략"의 표현법(비전투 트리거·자발 교체·사거리 밖). 게이트도 같은 패턴 — 함수 맨 앞에
  게이트 체크를 두고, 막히면 targets 비우고 return. per-target 루프가 비어 데미지 0(턴 손실).
- **지속/만료 기계**: active_states의 `expire_trigger`+`expire_count`. `_notify_event`가
  이벤트 발생마다 매칭 상태의 count를 1 차감, 0 이하면 제거(L205~212). TURN_END 브로드캐스트가
  `ON_TURN_END` 이벤트로 active_char 상태를 차감(L228·234). → 게이팅 상태를
  `expire_trigger:"ON_TURN_END"`, `expire_count:turns`로 두면 게이팅 턴마다 1차감, 정확히
  N턴 후 자동 소멸(고정 N턴, 사용자 결정).
- **RNG**: `StochasticityModule`이 시드 제어(`self.rng`)되나 범용 확률 roll이 없음 →
  base에 `roll_chance(p)` 추가. 모든 서브클래스(NoVariance 기본 포함) 상속 → 마비/혼란 판정이
  시드 재현(백테스트 결정론).
- **부여 경로**: `_act_move_effect`(ctx 받음)에 `kind:"inflict_status"` 분기 추가. 해저드/날씨
  무브와 동일 패턴. 대상(scope)의 active_states에 게이팅 상태 append.

## 3. config 스키마 (정적, 병렬 안전)
`game_config["mechanisms"]["status_gates"]` = 게이팅 상태 *정의*. weather_defs와 동형.
```
{
  "paralysis": {"gate": "skip_chance", "chance": 0.25, "turns": 0},                  # 영구(turns 0)
  "sleep":     {"gate": "skip_full",   "turns": 2},                                  # 2턴 완전 행동불가
  "confusion": {"gate": "confuse",     "chance": 0.5, "turns": 3, "self_hit_percent": 0.125}
}
```
gate 종류:
- `skip_chance` — 매 턴 `chance` 확률로 행동불가. turns 0이면 영구, >0이면 N턴 후 소멸. (마비)
- `skip_full`   — 상태 지속 동안 무조건 행동불가. (잠듦)
- `confuse`     — 매 턴 `chance` 확률로 자해(주 자원 max × `self_hit_percent`) + 행동불가. (혼란)

동적 상태(현재 누가 무슨 게이팅에 걸렸나)는 캐릭터 active_states에 산다(전투 범위). 정의는
game_config 정적(병렬 안전) — substrate 분리 원칙과 동일.

## 4. 사용자 결정 (확정)
- (a) **혼란 자해 데미지** = config 고정 비율(`self_hit_percent` × 주 자원 max). 데미지 공식
  비의존·회귀 안전·해저드/날씨 chip과 일관. (결정: 고정 비율)
- (b) **잠듦/혼란 지속** = 고정 N턴(`turns` 설정값, 기존 expire_count 기계 재사용). 결정론·
  백테스트 깔끔. (결정: 고정 N턴)
- (엔지니어링 콜, 권고대로) RNG는 roll_chance(시드 재현). 마비 속도감소는 별개(기존 스탯
  디버프 move_effect로 충분 → 게이트는 행동차단만). 만료는 ON_TURN_END expire_count 재사용.

## 6. 만료 타이밍 추적 (정확성 근거)
잠듦 turns=2, 상대 O에게 부여(공격자 턴):
- O 턴1: STAT_CALC는 ON_TURN_START라 무차감. 게이트가 상태(count 2) 봄 → skip. 턴종료
  TURN_END → count 2→1.
- O 턴2: 게이트가 상태(count 1) 봄 → skip. 턴종료 → 1→0 → 제거("소멸" 로그).
- O 턴3: 상태 없음 → 행동.
= 정확히 2턴 행동불가. 마비(turns 0)는 PERMANENT라 영구, 매 턴 chance roll.

## 7. 정직한 한계
- 게이팅은 싱글 액티브 가정 — active_char 1명 기준(멀티는 갈래 ④).
- 자해로 진입 KO 시 정리는 다음 _resolve_faint 사이클(갈래 ⑥과 동일 근사).
- 상태 중첩(마비+혼란 동시)은 active_states에 둘 다 들어가며 게이트가 순서대로 검사 —
  첫 막힘에서 return(스택은 자연 처리, 단 "어느 게 먼저"는 리스트 순서).
- "특정 게임 마비 확률값" 복제는 config 입력 — 엔진은 메커니즘만 1급 표현.
