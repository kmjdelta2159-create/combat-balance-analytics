# 동적 메커니즘 PR-E — 무브 효과 경로 (boost 무브)

Antigravity 작업 지시서. 전략 의사결정 모델의 선행 조건이다. 무브가 데미지 외의 효과(스탯 boost·디버프 등)를 낼 수 있는 길을 연다 — 이게 있어야 "boost를 깔까, 때릴까" 같은 진짜 결정이 생긴다.

새 per-target hook `ON_MOVE_EFFECT`를 ON_MOVE_USE 직후·DAMAGE_CALC 이전에 도입한다. 선택된 무브 이름이 `game_config["move_effects"]`에 정의돼 있으면, 그 효과(스탯 boost)를 기존 active_states에 영구 부여한다(scope=self 사용자, target 타겟). get_effective_stat이 이미 active_states를 합산하므로 boost가 즉시 반영된다. move_effects 미설정 시 no-op이라 회귀가 없다.

## move_effects 스키마

`game_config["move_effects"][<무브이름>] = [ {"target_stat","mod_type","value","scope"}, ... ]`. 리스트라 Calm Mind처럼 한 무브가 여러 스탯을 boost할 수 있다(SpAtk+SpDef). mod_type은 "percent"(1.0=+100%) 또는 "flat", scope는 "self"|"target". 영구(PERMANENT) 상태로 부여되어 사용할 때마다 누적(스택)된다.

## 범위

`modules/engine.py` 4곳만. 이번 PR은 엔진 경로다 — 효과를 game_config로 주입하면 작동하며 하니스로 검증된다. 사용자가 무브에 효과를 붙이는 입력 UI는 다음 PR(PR-E2)이다. 그리디 선택기는 여전히 위력 0 boost 무브를 안 고르므로, 전략적 선택은 PR-F(정책)가 담당한다 — 이 PR은 "효과 무브를 지정·강제하면 효과가 적용됨"까지를 연다.

## 적용 규칙

- FIND를 찾아 REPLACE로 그대로 교체한다. 들여쓰기·공백·한글 한 글자도 바꾸지 않는다.
- 임의 리팩터·이름 변경·하드코딩 금지. 지시된 치환만.
- `_act_move_effect`는 기존 `_track_state`(같은 파일에 정의됨)를 호출한다 — 새로 만들지 말 것.
- 적용 후 `python -c "import modules.engine"` 통과.

```python
def _act_move_effect(ctx):
    """무브 사용 시 효과(스탯 boost 등) 적용 — game_config['move_effects'] 기반.
    선택된 무브 이름에 효과가 정의돼 있으면 active_states에 영구 boost를 부여한다.
    scope='self'면 사용자, 'target'이면 현재 타겟. 미정의 시 no-op."""
    move = ctx.get("current_move")
    if not move:
        return
    effects = (ctx.get("game_config") or {}).get("move_effects") or {}
    spec_list = effects.get(move.get("name"))
    if not spec_list:
        return
    char = ctx["active_char"]
    tgt = ctx.get("current_target")
    applied = False
    for spec in spec_list:
        recipient = char if spec.get("scope", "self") == "self" else tgt
        if recipient is None:
            continue
        _track_state(ctx, recipient, {
            "id": f"move_effect_{move.get('name','?')}_{spec.get('target_stat','?')}",
            "target_stat": spec.get("target_stat"),
            "mod_type": spec.get("mod_type", "percent"),
            "value": float(spec.get("value", 0)),
            "expire_trigger": "PERMANENT",
            "expire_count": 9999,
            "source_id": char.get("id"),
        })
        applied = True
    if applied:
        ctx["add_log"](
            f"  -> [Phase: ON_MOVE_EFFECT] {char.get('id','?')} {move.get('name','?')} 효과 적용"
        )


# ── 디폴트 액션을 글로벌 레지스트리에 등록 ──
```
