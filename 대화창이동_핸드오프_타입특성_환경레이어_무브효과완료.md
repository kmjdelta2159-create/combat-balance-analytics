# 대화창 이동 핸드오프 — 타입/특성(D1) + 효과 디스패처 복원·환경 레이어 완성(E′1~E′2f) + 무브효과(M1 Explosion·M2 Trick)

새 대화창에 이 문서를 붙여넣고, 직전 핸드오프(`대화창이동_핸드오프_풀배틀리플레이resync완료.md`)와 함께 읽어라. 본 문서는 그 위에서 **잔차 버킷을 데이터로 하나씩 닫는 역설계→수정 루프를 전투 스케일로 길게 돌린** 결과를 기록한다. 역할·제약·방법론은 그대로다: **코드 직접 수정 금지·Antigravity 프롬프트(.md) 산출·검증 의무·시간 추정 금지·시연 언급 금지·도구 위치 과대표현 금지·말투 prose 우선·갈래는 항상 최종목표 기준으로 사용자에게 묻고 결정받기.**

## 0. 한 줄 요약 (현재 위치)

골든 gen5 싱글의 라운드별 데미지 충실도가 **구조적으로 ~75~80%**까지 닫혔다. 키스톤(효과 디스패처)이 유실됐던 것을 복원·확장해 **5 소스(ability/item/status/weather/move) × 4 효과타입(damage_frac[누진 포함]·heal_frac·self_faint·swap_item) × 조건(contact/weather/of_types/not_types/not_ability)**을 갖췄고, 환경 레이어(회복·날씨·상태·맹독·Poison Heal)·무브효과(Explosion 자폭·Trick 도구교환)·타입특성(HP Fire·Levitate)이 전부 *데이터*로 얹혔다. 남은 건 작거나 점근적인 잔차뿐.

## 1. 이번 phase 구축물 (적용·검증 완료, 순서대로)

전부 한글 프롬프트 .md 직접 작성 + 클린룸 검증 + 사용자 앱 run_b4 대조. **모두 적용·작동 확인됨.**

* **PR-D1 — 타입/특성** (`풀배틀리플레이_PR-D1_타입특성_프롬프트.md`): **엔진 무변경**(키스톤 발견 전). (a) Hidden Power 숨김타입을 trace-action 생성 시 공격자 세트 `hp_type`(Rotom-Wash·Latios = Fire)로 해석 + **TYPE에 Fire 행 추가**(없으면 ×1 무력화 — 클린룸이 잡음). (b) Levitate 등 특성 타입면역을 *의사-타입 컬럼 t3* + `type_table[공격타입][특성]=0.0`으로 데이터화(`ABILITY_TYPE_IMMUNITY`). **결과: T1 Rotom-W HP Fire 0.93 닫힘, T16 Garchomp EQ→Latios 엔진 −253→0(Levitate 면역).** reference_gen5 + battle_setup만.
* **★발견 — 키스톤(효과 디스패처)이 엔진에 없었다**: 직전 핸드오프가 "PR-E1/E2 디스패처·F3 날씨 적용"이라 했으나 현재 engine.py엔 디스패처도 weather도 없고(Grep 0건), 단발 슬롯(leftovers/status/protean/hazard) + move_effects만 있었다. battle_setup이 `mechanisms['effects']`를 주입하나 소비자가 없어 *배선이 끊겨* 있었다(B3~C2 정리 중 유실 추정). 설계안: `환경레이어_효과디스패처복원_설계안.md`.
* **PR-E′1 — 디스패처 코어 복원 + ON_HIT 접촉** (`환경레이어_PR-E1_디스패처복원_프롬프트.md`): 유실된 `_act_effect_dispatch`/`_eff_*` 복구 + `_act_on_hit` 끝 호출. reference EFFECTS(dict)·CONTACT_MOVES + battle_setup가 char에 ability/item 노출. **세트 역설계: Garchomp = Rough Skin + Rocky Helmet**(env −52=420/8·−70=420/6, prior Sand Veil/Life Orb 오류). **T5 Hippowdon 접촉반동 0→−122.**
* **PR-E′2a — 턴엔드 heal_frac** (`..._PR-E2a_턴엔드회복_...`): 디스패처 ON_TURN_END 등록(페이즈별 owner — 턴엔드는 active_char만, 이중회복 가드) + heal_frac + of_types. **Leftovers(Hippowdon·Zapdos·Jirachi·Ferrothorn)·Black Sludge(Tentacruel, Poison)** 세트 역설계. **T5 −96/−96 정확·T10/T18/T24/T25 Leftovers 닫힘.**
* **PR-E′2b — 날씨/상태** (`..._PR-E2b_날씨상태_...`): 날씨 토큰 키 + weather 조건 + build_ctx가 `field_state['weather']`를 `weather_by_turn`(트레이스 도출)에서 공급 + resync가 status 공급. **모래칩(not_types 면역)·Rain Dish·burn.** **T17 Latios 모래 −19/−19 닫힘.**
* **PR-E′2c — 상태 캡처 보강** (`..._PR-E2c_상태캡처_...`): build_snapshots가 switch에서만 잡던 status를, showdown_trace의 `-status`/`-curestatus` 이벤트 emit + build_snapshots 추적으로 보강(burn/tox/par 캡처). **burn unblock → T9 Rotom-Wash −33/−33.** 엔진 무변경(순수 파이프라인).
* **PR-E′2d — 맹독 누진** (`..._PR-E2d_맹독누진_...`): damage_frac에 progressive(엔진 char `tox_stage` 카운터, 턴엔드마다 증가) + resync가 off-field/비-tox 시 리셋. forced 진입 유닛은 턴엔드 없어 무틱(자연 정합). **env 레이어 1차 완성.**
* **PR-E′2e — 상태 공급 타이밍** (`..._PR-E2e_상태공급타이밍_...`): 진단(run_tedge_diag)으로 맹독 stage가 1턴 밀림을 확정 — resync가 status를 log[T−1]에서 줘서 *부여 턴 틱*을 놓침. **status를 log[T]에서 공급**(HP·on_field는 log[T−1] 유지). **T7 Hippowdon −26/−26 닫힘.**
* **PR-E′2f — Poison Heal** (`..._PR-E2f_PoisonHeal_...`): E′2e가 드러낸 새 잔차 — **Breloom = Poison Heal + Toxic Orb**(T1 Spore+자기맹독·맹독데미지無). `not_ability` 조건으로 Poison Heal 보유자의 맹독 데미지 억제. **T1 Gengen −275→−256 복귀.**
* **PR-M1 — Explosion 자폭** (`무브효과_PR-M1_Explosion자폭_...`): 디스패처 **무브-소스**(ON_HIT에 사용자 무브명을 키로) + **self_faint** 효과타입. 대상 데미지는 엔진이 이미 정확(T15 Bonaparte −108/−108)이라 사용자 HP→0만. **T15 Ferrothorn −282/−282·T19 Metagross −323/−323 닫힘**(자폭 fainted).
* **PR-M2 — Trick 도구교환** (`무브효과_PR-M2_Trick아이템교환_...`): **swap_item** 효과타입 — active↔target item 교환 + `item_stat_mults`(build_game_config=ITEMS)로 정적 스탯배율 보정. **T20 Latios↔Zapdos(Choice Specs↔Leftovers).** Latios spa 538→358(Specs 제거) → **T22 −175→−109**(로그 −93, 롤 수준) + Latios Leftovers 회복 동시 닫힘.

진단 도구: `run_tedge_diag.py`(턴엔드/상태 공급을 행동 유닛별로 떠보는 앱사이드 진단 — 맹독 1턴 지연·Poison Heal을 이걸로 짚음).

## 2. 핵심 진단 — 역설계 루프가 세트·메커니즘을 데이터로 지목

* **키스톤 부재 발견**: 직전 핸드오프 서사와 실제 엔진이 갈렸다(디스패처·날씨 유실). 데이터(run_b4 ★ 다수가 "엔진 0")가 환경 레이어 미배선을 드러냄 → 디스패처 복원이 진짜 토대였다.
* **env 스트림이 세트를 역설계**: Garchomp Rough Skin+Rocky Helmet, Tentacruel Black Sludge, Breloom Poison Heal+Toxic Orb, Leftovers 보유자 4종, Latios↔Zapdos Trick 스왑 — 전부 잔차/env가 가리킨 뒤 인코딩 → 닫힘.
* **상태 공급 타이밍이 핵심**: 무브-부여 status(Hippowdon 맹독)는 부여 턴에 틱하므로 log[T], 도구-부여(Breloom Toxic Orb)는 턴 끝 발동이라 별개. log[T] + Poison Heal 억제로 둘 다 맞춤. (정밀 일반해는 엔진이 무브-부여 status를 턴 중 적용하는 것 — 미구현.)
* **resync가 desync 흡수 + 라운드별 격리**를 계속 보장. 마지막 캡처 턴 = 27 = 로그 27 유지(전 PR 회귀0).

## 3. 현재 진행도 (정직)

* **하니스(재사용 기계): 사실상 완료.** 27턴 resync 재생 + 라운드별 divergence 격리.
* **이 골든 gen5 전투 수치 충실도: ~75~80%.** 데미지 셀 35개 중 구조적 ★ 8~10개 잔존, 나머지 닫힘/롤. 닫힌 버킷: 타입/특성·접촉반동·환경 전부(회복·날씨·상태·맹독누진·Poison Heal)·Explosion·Trick·상태타이밍.
* **일반화(임의 싱글·gen6·비-포켓몬): 낮음.** gen5 한 판 + gen6 가벼운 교차에만 단단. 비-포켓몬 가드 미실시 — 1차 목표("임의 싱글")의 상한.
* **점근적 꼬리**: 데미지 롤(±15%, 로그 미관측)·세트/EV 정밀화는 원리상 100% 불가. 실질 목표 = *구조적 divergence가 롤·세트 노이즈로 수렴*.

## 4. 남은 잔차 버킷 (다음 타깃 — 점점 작거나 점근적)

## 5. ★운영 — mnt 절단 상존(이번 세션 핵심 함정)

샌드박스 mnt 마운트가 **최근 수정된 .py 파일**(Antigravity 적용분·내 Write 직후)을 *끝부분에서 잘라 읽는다*(bash/python/cp/ast.parse 전부). 미수정 모듈·HTML·트레이스는 정상. 그래서:
* **Read/Grep이 진실 출처** — 적용분 대조는 Read/Grep으로(엔진 전체 실행은 앱사이드 run_b4).
* **클린룸은 /tmp 재구성** — mnt에서 modules를 /tmp/proj로 cp하면 수정 파일은 잘리니, 필요한 작은 모듈은 Read 내용으로 /tmp에 재기록하거나 인라인. showdown_trace·fullbattle_diff 등은 cp로 정상(미수정 시).
* **프롬프트 작성 후 wc/tail로 무결성 확인** — Write가 mnt에 잘려 들어가는 일이 있었음(verify 스크립트 1회). 단, 워크스페이스 .md는 대체로 온전했고 사용자 앱은 실제 파일을 읽음.
* per-event는 run_resid(엔진 무관, 단 reference_gen5 절단 시 import 불가 → 인라인), 풀런은 run_b4(앱사이드).

## 7. 방법론 — 이번 phase 확인·강화

* **디스패처 = 키스톤이 실증됨**: 새 메커니즘 = EFFECTS 한 줄 + (필요 시) 효과타입/조건 1개. Explosion·Trick·Poison Heal이 그렇게 들어감. 소스가 ability→item→status→weather→move로 자라며 owner-키 루프를 재사용.
* **잔차가 세트를 데이터로 지목**: 한 잔차를 닫으면 다음을 드러낸다(E′2e→Poison Heal). prior 세트는 추정, env/잔차가 교정 타깃을 정함.
* **클린룸 우선 + 앱 대조**: 모든 효과·산식을 적용 전 standalone으로 검증(산수 일치까지), 적용 후 Read 대조 + run_b4 닫힘 확인. 회귀0 게이트(미설정 no-op) 모든 PR 유지.
* **진단 먼저, 추측 금지**: 턴엔드 누락을 "상대 교체" 가설로 추측했으나 run_tedge_diag가 진짜 원인(상태 1턴 지연·Poison Heal)을 짚음. 추론으로 못 좁히면 런타임 로그를 떠라.
* 갈래는 최종목표 기준 사용자 결정. 시간 추정·시연 언급·도구 위치 과대표현 금지. 한글 프롬프트 .md 직접 작성. 단계별 보고 + 사용자 결정.
